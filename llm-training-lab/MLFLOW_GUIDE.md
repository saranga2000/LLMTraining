# MLflow Experiment Tracking Guide

> How to launch, navigate, and interpret the MLflow UI for this training lab.

---

## Launching MLflow

```bash
cd llm-training-lab
source venv/bin/activate

# MLflow 3.x has strict security middleware — disable it for local use
mlflow server \
  --backend-store-uri ./mlruns \
  --host 127.0.0.1 \
  --port 5000 \
  --disable-security-middleware &

# Open in browser (use 127.0.0.1, NOT localhost — avoids 403 errors)
open http://127.0.0.1:5000
```

**Troubleshooting:**
- If you get a 403 error, use `127.0.0.1` instead of `localhost`
- If you see a blank page, open an incognito window (browser may cache old 403)
- The `--disable-security-middleware` flag is safe for local-only servers

## Inside Each Run

Each experiment has one run. Click the **run name** (e.g., `nanogpt_v1`) to open it.

### Parameters Tab — The "Recipe Card"

This is where every hyperparameter is logged. It makes the experiment **fully reproducible** — anyone can re-run with identical settings.

**Phase 1 (`nanogpt_v1`):**

### Metrics Tab — The Learning Curves

Click any metric name to see its chart over training time.

**Key metrics to look at:**

- **`train_loss`** — The main learning signal. Should decrease over time.
  - Sharp early drop = model is learning fast
  - Flattening = diminishing returns, model has captured most patterns
  - If it spikes = training instability (didn't happen here)

- **`eval_loss` (or `val_loss`)** — Performance on held-out data the model never trained on.
  - Should track close to train_loss = healthy training
  - If eval_loss rises while train_loss falls = **overfitting** (memorizing, not generalizing)
  - We didn't see overfitting in any phase — good sign

**What the loss numbers actually mean:**

| Loss Value | Perplexity (e^loss) | Interpretation |
|-----------|-------------------|----------------|
| 4.2 | 66.7 | Random guessing (Phase 1 start) |
| 2.4 | 11.0 | Knows common patterns |
| 1.8 | 6.0 | Reasonably good predictions |
| 1.3 | 3.7 | Confident and mostly correct |
| 1.0 | 2.7 | Very fluent |

Perplexity = "how many equally-likely options the model sees at each step." Lower = more confident and correct.

## The Story Across All Three Experiments

This is the most important thing to see — how each phase builds on the last:

| Experiment | Starting Loss | Final Train Loss | Final Eval Loss | What Happened |
|-----------|--------------|-----------------|-----------------|---------------|
| `pretrain_shakespeare` | ~4.2 | ~1.48 | ~1.57 | Learned English from zero (character-level) |
| `sft_dolly` | ~2.7 | ~2.39 | ~2.25 | Learned to follow instructions |
| `lora_medical` | ~1.8 | ~1.30 | ~1.33 | Learned medical domain knowledge |

**Key insight:** Each phase starts at a lower loss because it inherits knowledge from the previous phase. This is the entire modern LLM training pipeline:

```
Random weights → Pretrain (learn language) → SFT (learn instructions) → LoRA (learn domain)
   loss ~4.2          loss ~1.5                  loss ~2.3                  loss ~1.3
```

The SFT loss is higher than pretrain because it's a different task (instruction-following is harder than character prediction), but the model starts from pretrained weights so it doesn't begin at random.

## Comparing Runs

MLflow has a **Compare** feature:
1. Go to an experiment
2. Select multiple runs (checkboxes)
3. Click "Compare"
4. See parameters and metrics side-by-side

This is useful if you run multiple experiments with different hyperparameters (e.g., LoRA with r=8 vs r=16 vs r=32).

---

## Why Experiment Tracking Matters

Without MLflow, you'd have scattered terminal outputs, forgotten hyperparameters, and no way to reproduce results. With it:

1. **Reproducibility** — Every parameter is logged. Anyone can re-run your exact experiment.
2. **Comparison** — Side-by-side metrics across runs with different settings.
3. **History** — When you try 10 different learning rates, you can see which worked best.
4. **Artifacts** — Generated samples are saved alongside metrics, not in random text files.

In production ML teams, this is how models are tracked from experimentation through deployment.
